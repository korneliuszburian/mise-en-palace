# KRN Failure Report

## Archived Direction

The old KRN direction is archive/research quarry only. Do not copy its topology
or continue its goal chain.

## Reusable Positive Evidence

- KRN as Codex Operating Layer.
- Thin `AGENTS.md`.
- Source-to-decision mapping.
- Store-backed Memory Core boundary.
- Bounded context packet.
- Pre-edit engineering gate.
- Review capture.
- Schema-backed contracts.
- Known-bad fixtures.
- Recognition that dashboard and benchmark can fake progress.

## Failure Modes

### Artifact Factory

The repo produced many goal files, reports, ledgers, memory notes, benchmark
lanes, dashboard view models, and status reports without proving the daily
workflow:

```text
task -> context packet -> Codex run -> diff/tests/review -> memory/source/eval feedback
```

### File-Backed Pseudo-Memory

Markdown notes looked inspectable but were not runtime memory. Runtime memory
must be queryable, temporal, source-linked, confidence-aware, TTL-aware,
invalidatable, feedback-aware, and retrievable as small packets.

### Eval Theater

Green evals do not prove product quality. Evals matter only when they protect a
real contract, known failure, golden task, security boundary, memory behavior,
source behavior, context behavior, review burden, or diff risk.

### Goal As Product Brain

Goal files must not absorb product thesis, source map, runtime instructions,
progress ledger, architecture backlog, and verification evidence. `GOAL.md` in
this repo is only the current bootstrap execution contract.

### Source Hoarding

Sources without mechanisms and decisions become decoration. Every retained
source needs `doesNotProve`.

## Hard Stops

Stop and re-scope if the next step requires:

- reading all historical goals;
- copying old repo structure;
- dashboard before typed objects;
- benchmark lane before dogfood traces;
- memory in markdown as runtime;
- many subagents before output contracts;
- writing code before pattern selection;
- adding evals without known failure/contract;
- broad research without hypothesis and budget;
- TypeScript weakening to move faster.
