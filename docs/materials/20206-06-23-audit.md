MemoryCandidate i MemoryReviewGate są dobrymi abstrakcjami. Gate sprawdza candidate status, lineage, guidance, confidence, temporal invalidation rule i source-claim existence, a CLI promotion wymaga `--evidence-reviewed-ref` .

Observation jest valid KRN mechanism, jeśli zostaje stagingiem: source-ranged, dated, temporal, nie-final-truth . Reflection też jest valid, ale tylko jako explicitly invoked candidate/reporting layer, nie autonomous dreaming/self-healing loop .

Największe ryzyko: ktoś przeczyta “brain”, “dreaming”, “reflection” i zacznie budować magiczną pętlę, która tworzy prawdę bez review.

## 6. Activation jest najważniejszą prawdziwą warstwą, ale ma metadata debt

Activation jest centralne: wybiera bounded context z memory/source/search/anti-memory/trust/temporal/ROI signals i umie abstain when context is weak . To jest serce KRN, bardziej niż `krn audit`, dashboard czy Promptfoo.

Ale część behavior-governing identity nadal żyje w metadata. To trzeba przenieść do typed fields/read models.

## 7. SourceGraph jest dobry, ale project scoping musi zostać uszczelniony

SourceGraph jako idea jest dobra: mechanism, implication, does-not-prove, consumer, trust, support type, rejection, decision links, falsifiers . Ale jeśli project scope siedzi w metadata albo jest pośrednio odtwarzany, to prędzej czy później dostaniecie global claim soup.

## 8. Promptfoo jest adapterem, nie dowodem

GoldenTask może zostać canonical KRN behavior contract. Promptfoo może być bounded external runner/result adapter. Current smoke proves integration only . Jeśli jakikolwiek doc mówi “Promptfoo pass = memory działa”, to doc kłamie.

## 9. Workers są skeletonem, nie runtime

Worker package może zostać jako job contracts. Nie może być opisywany jako realne dreaming/maintenance runtime. Reset-audit mówi dokładnie: acceptable as bounded contract, but not running background maintenance .

Obecnie executor może wejść przez README, GOAL, root PLAN, memory PLAN, handoff, audit docs, raw materials i dostać różne “current truths”. To jest śmiertelne dla KRN, bo KRN ma rozwiązywać context selection, a samo repo już na poziomie dokumentów generuje context poison.

* local DB is unavailable for DB-required slice;
* typecheck/test cannot run;
