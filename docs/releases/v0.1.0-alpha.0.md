# KRN v0.1.0-alpha.0

Status: internal-alpha tag for G-02.

Date: 2026-06-25

## Install

```sh
git clone https://github.com/korneliuszburian/mise-en-palace.git
cd mise-en-palace
git checkout v0.1.0-alpha.0
pnpm install --frozen-lockfile
pnpm alpha:verify
```

## Operator Command

Use:

```sh
pnpm krn ...
```

Examples:

```sh
pnpm krn doctor
pnpm krn plan --task "inspect target repo"
```

## Included Foundations

- DB-backed KRN planning/evidence/observe/reflect dogfood path.
- Evidence dirty-context classification.
- Candidate reviewability output.
- Run readback JSON.
- CI verification pipeline.
- Local brain-store backup/restore policy.
- Internal-alpha source workspace install path.

## Required Local Verification

```sh
pnpm alpha:verify
pnpm db:ready
pnpm db:smoke
```

DB commands require local Postgres/pgvector from `compose.yaml`.

## What This Proves

- The tag points to a repeatable source workspace install path.
- Operators can use root `pnpm krn` instead of package-internal commands.
- CI and local verification commands can prove the checked-in source state.

## What This Does Not Prove

- product readiness;
- npm publication readiness;
- global binary install readiness;
- hosted production deployment;
- dashboard/API/MCP/plugin readiness;
- worker runtime execution;
- target-repo internal-alpha readiness.

## Known Limitations

- Packages remain `private: true`.
- CLI runs through pnpm/tsx workspace tooling.
- DB-backed operation requires local Postgres/pgvector.
- Production DR/PITR is not implemented.
- G-03 must still decide internal-alpha readiness from target-repo evidence.
