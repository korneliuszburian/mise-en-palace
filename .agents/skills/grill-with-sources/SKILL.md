# Grill With Sources

Use this skill before planning when the problem is not yet sharp enough to
implement.

## Workflow

1. Restate the problem in precise KRN vocabulary.
2. List assumptions and mark which are sourced, local, or user-provided.
3. Identify conflicting mechanisms or incentives.
4. Separate hard facts from interpretation.
5. Build decision options with tradeoffs.
6. Build a rejection list.
7. Name the missing evidence that would change the decision.

## Output

- Vocabulary.
- Assumptions.
- Conflicts.
- Source-backed mechanisms.
- Decision options.
- Rejections.
- Missing evidence.
- Recommended next planning or implementation step.

## Forbidden

- Do not jump to implementation before shared understanding.
- Do not turn source review into bibliography.
- Do not simplify terms until engineering meaning is lost.
- Do not expand into broad research without a fixed question and budget.

## Verification

The result should make a later implementation plan decision-complete or expose a
specific blocker.
